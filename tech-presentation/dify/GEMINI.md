# Dify 技術深潛 — Presentation Project

This project is a [Slidev](https://sli.dev/) presentation titled **"Dify 技術深潛 — RD 分享"** (Dify Technical Deep Dive - RD Sharing). It covers the architecture, core nodes, RAG details, community issues, and best practices for the Dify open-source agentic workflow platform.

## Project Structure

- `slides.md`: The main entry point for the presentation content.
- `package.json`: Project configuration and scripts.
- `components/`: Custom Vue components used in the slides.
    - `SectionSlide.vue`: Layout for section header slides.
    - `InfoCard.vue`, `IssueCard.vue`, `Callout.vue`: UI components for displaying information and GitHub issues.
- `pages/`: Additional slide content (e.g., `imported-slides.md`).
- `snippets/`: Code snippets used within the slides.
- `dify-notes.md`: Comprehensive background notes and technical details about Dify.
- `netlify.toml` / `vercel.json`: Deployment configurations for Netlify and Vercel.

## Key Technologies

- **[Slidev](https://sli.dev/)**: Presentation tool for developers.
- **Vue 3**: Framework for custom components.
- **UnoCSS**: Atomic CSS engine (configured via `slides.md` frontmatter).
- **Shiki**: Syntax highlighter for code blocks.
- **Bun**: Fast JavaScript package manager (used via `bun.lock`).

## Getting Started

### Prerequisites

- [Bun](https://bun.sh/) or [Node.js](https://nodejs.org/)
- [npm](https://www.npmjs.com/) (if not using Bun)

### Commands

| Command | Description |
|---------|-------------|
| `bun run dev` / `npm run dev` | Start the development server and open the browser. |
| `bun run build` / `npm run build` | Build the presentation for static hosting (outputs to `dist/`). |
| `bun run export` / `npm run export` | Export the presentation to PDF. |

## Development Conventions

- **Content Editing**: Most changes should be made directly in `slides.md`.
- **Custom Components**: New UI elements for slides should be added to the `components/` directory as Vue 3 SFCs (Single File Components).
- **Styling**: Use UnoCSS classes (Tailwind-compatible) for styling within slides and components.
- **Theming**: The presentation uses the default Slidev theme with a dark color scheme.

## Deployment

The project is configured for easy deployment to:
- **Netlify**: Using the `netlify.toml` configuration.
- **Vercel**: Using the `vercel.json` configuration.
- Both target the `dist/` directory generated by `npm run build`.
